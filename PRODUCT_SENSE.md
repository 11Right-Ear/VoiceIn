# PRODUCT_SENSE.md — 产品感文档

## 产品定义

**VoiceIn** 是一个极简的中文语音输入工具。用户按下快捷键，说话，文字自动出现在光标所在位置。

一句话描述：**"说话即输入"——把你的语音变成光标处的文字。**

---

## 核心问题

### 为什么需要这个工具？

1. **中文输入慢**：拼音输入需要逐字选词，手写更是低效。语音输入的速度是打字的 3-5 倍
2. **现成方案太重**：
   - 讯飞输入法：附带整个输入法框架，200MB+
   - 系统自带语音输入（Win+H）：识别率一般，体验割裂
   - 手机语音输入方便，但回到桌面端就没了
3. **写作者、笔记党、聊天族**：有大量"想说的话"需要快速变成文字的刚需

### 核心价值主张

> 比你拿起手机用语音输入法更快、比你打字更轻松。

- **零界面**：没有窗口，没有 UI 学习成本
- **零步骤**：按键 → 说话 → 文字出现，三步合一
- **零干扰**：不切换窗口，不离开当前工作流

| 角色 | 场景 | 痛点 |
|------|------|------|
| 写作者 | 写文章、日记、笔记 | 打字打断思路流，语音更自然 |

| 长辈用户 | 不太会打字 | 语音是唯一自然的输入方式 |

## 设计原则

### 原则 0：极简第一

> 任何功能增加 1% 的复杂度，必须带来 10% 的用户价值。

- 一个功能如果能"默认做好"就不给用户选项
- 代码量、内存、安装包体积都要严格克制
- 不做配置 UI——配置文件即界面

### 原则 1：不打断用户

- 不在屏幕上弹窗（用托盘图标状态表示）
- 不在粘贴后发出声音或震动
- 识别失败时静默处理，不弹错误框

### 原则 2：键盘优先

- 全局热键是唯一的触发入口（保留托盘点击作为备选）
- 粘贴后焦点回到原来的窗口和应用
- 整个过程手不离开键盘

### 原则 3：默认就是最好

| 决策 | 默认值 | 不给选项的理由 |
|------|--------|---------------|
| 麦克风 | 系统默认 | Windows 用户已在系统设置中选好 |
| 识别语言 | 中文 | 产品定位就是中文输入 |
| 粘贴行为 | 自动粘贴 | 核心价值就是少操作 |
| VAD 超时 | 1.5 秒静音自动停止 | 用户不用手动停止 |

## 核心交互流程

```
用户正在打字/写代码/聊天...
         │
         ▼
  按下 Ctrl+Shift+V
         │
         ▼
  [托盘图标变红] ← 视觉反馈"正在听"
         │
         ▼
    开始说话...
         │
         ▼
  说完（静音 1.5 秒 / 再次按热键）
         │
         ▼
  [托盘图标恢复] ← 录音结束
         │
         ▼
  文字出现在光标处 ← 自动粘贴完成
```

## 功能边界

### v1 必须做的

- [ ] 全局热键触发/停止录音
- [ ] WASAPI 麦克风采集（C++）
- [ ] Sherpa-ONNX 本地中文流式识别
- [ ] 自动粘贴到当前焦点窗口光标处
- [ ] 系统托盘图标 + 状态指示
- [ ] 首次运行自动下载模型

### v1 明确不做的

- [ ] 云端 ASR（v2）
- [ ] GUI 设置窗口——配置文件就是设置
- [ ] 多语言支持——只做中文
- [ ] 历史记录——不存任何录音或文本
- [ ] 自动更新——手动下载新版本
- [ ] 用户登录——单机工具
- [ ] 离线模型管理 UI——一个模型走天下
- [ ] 标点自动添加——v2

### v2+ 可考虑的

- 云端 ASR fallback（讯飞 API）
- 标点智能添加
- 热键自定义
- 特定领域词库（代码术语、医学术语等）
- iOS / Ubuntu 移植

## 竞品对比

| | VoiceIn | 讯飞输入法语音 | Win+H | 剪映字幕 |
|---|---|---|---|---|
| 启动方式 | 全局热键 | 切到讯飞输入法 | Win+H 快捷键 | 打开剪映 |
| 界面干扰 | 零（仅托盘） | 输入法浮窗 | 全屏覆盖层 | 视频编辑器 |
| 安装体积 | <100MB | >500MB | 系统自带 | >1GB |
| 离线可用 | 是 | 部分 | 否 | 否 |
| 中文准确率 | 良好（本地模型） | 优秀（云端） | 一般 | 优秀（云端） |

## 产品成熟度路线

```
v1.0 — Windows 本地识别 MVP
       ├── WASAPI 采集 + Sherpa-ONNX 识别
       ├── 托盘 + 热键 + 自动粘贴

v1.1 — 体验打磨
       ├── 静音检测优化
       ├── 热键冲突检测

v1.2 — 云端集成
       ├── 讯飞 API fallback
       └── 联网自动切换云端

v2.0 — 跨平台
       ├── Ubuntu (ALSA + Sherpa-ONNX)
       └── iOS (AVAudioEngine + CoreML)
```
