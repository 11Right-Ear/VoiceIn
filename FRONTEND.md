# FRONTEND.md — 前端/UI 设计文档

## 设计哲学

VoiceIn 的前端只有一个元素：**系统托盘图标**。

没有窗口、没有设置面板、没有确认对话框。所有交互通过托盘图标 + 全局热键完成。这是故意的——我们相信"最好的 UI 就是没有 UI"。

---

## 一、托盘图标

### 1.1 两种状态

| 状态 | 图标 | 含义 |
|------|------|------|
| 空闲 (IDLE) | 灰色麦克风 | 等待热键触发，准备好接收语音 |
| 录音中 (RECORDING) | 红色麦克风 + 外圈脉冲动画 | 正在收听，用户应说话 |

### 1.2 图标规格

- 尺寸：16×16 px（系统托盘标准）、32×32 px（高 DPI 备选）
- 格式：ICO 文件，内嵌两种尺寸
- 来源：代码内 base64 嵌入，不依赖外部图标文件

图标设计要点：
- 极简线条的麦克风轮廓
- 灰色（#888888）表示空闲，红色（#E74C3C）表示录音中
- 不需要文字——麦克风是通用的"语音"符号

### 1.3 工具提示 (Tooltip)

| 状态 | 显示文本 |
|------|----------|
| 启动 | "VoiceIn — Ctrl+Shift+V 开始语音输入" |
| 录音中 | "VoiceIn — 正在录音... 再次按 Ctrl+Shift+V 停止" |
| 下载模型 | "VoiceIn — 正在下载语音模型 (45%)" |
| 错误 | "VoiceIn — 麦克风不可用" |

Tooltip 是用户发现功能的主要途径，所以第一个 tooltip 必须显式告诉用户热键。

### 1.4 状态切换视觉过渡

```
IDLE 状态下按热键 →
  图标立即切换为红色麦克风（< 100ms）
  工具提示更新为"正在录音..."
  （可选：Windows 10+ 的托盘图标可触发微小的弹跳动画）

停止录音 →
  图标切换回灰色麦克风
  短暂闪烁一次（100ms 亮度变化）表示"动作完成"
  工具提示恢复
```

## 二、右键菜单

```
┌──────────────────────────┐
│  🎤 VoiceIn              │  ← 标题行，不可点击
├──────────────────────────┤
│  ✓ 中文识别              │  ← 当前语言（v1 仅中文）
├──────────────────────────┤
│  关于 VoiceIn            │  ← 弹 About 迷你窗口
│  退出                    │  ← 完全关闭程序
└──────────────────────────┘
```

### 菜单项说明

| 菜单项 | 行为 |
|--------|------|
| "中文识别" | 显示勾选，v1 不可更改。v2 可切英文 |
| "关于 VoiceIn" | 弹出极简 About 窗口（版本号 + 模型名 + GitHub 链接），窗口仅 300×200 |
| "退出" | 停止录音（如果正在录），释放 DLL 资源，退出进程 |

### 绝不在右键菜单中添加的内容

- 设置入口 → 配置文件就是设置
- 历史记录 → 产品不存历史
- 手动切换语言 → v1 只有中文
- 帮助/教程 → 热键信息在 tooltip 里

## 三、通知气泡

### 3.1 何时触发通知

| 场景 | 标题 | 内容 | 持续时间 |
|------|------|------|---------|
| 首次启动 | "VoiceIn 已就绪" | "按 Ctrl+Shift+V 开始语音输入" | 5 秒 |
| 下载模型 | "正在下载语音模型" | "首次运行需要下载 (XX MB)" | 跟随进度 |
| 下载完成 | "模型就绪" | "现在可以开始使用了" | 3 秒 |
| 无麦克风 | "未检测到麦克风" | "请插入麦克风后重启应用" | 手动关闭 |
| 启动时热键冲突 | "快捷键冲突" | "Ctrl+Shift+V 已被其他应用占用" | 5 秒 |

### 3.2 通知规则

- 不弹任何"识别成功"的通知（核心操作零干扰）
- 错误通知需要用户手动关闭的仅"无麦克风"
- 其他通知自动消失
- v1 不做通知历史

- 纯 Python tkinter 实现（Python 自带，零依赖）

## 五、用户体验状态表

| 时刻 | 用户动作 | 系统反馈 | 反馈延迟 |
|------|----------|----------|---------|
| 启动程序 | 双击 exe | 托盘出现灰色麦克风 → 气泡"已就绪" | < 500ms |
| 开始录音 | 按 Ctrl+Shift+V | 图标变红 | < 100ms |
| 录音中 | 说话 | 无视觉变化（图标保持红色） | — |
| 手动停止 | 再按 Ctrl+Shift+V | 图标闪一下 → 变灰 → 文字粘贴 | < 200ms |
| 自动停止 | 静音 1.5 秒 | 同上 | < 200ms + 1500ms VAD |
| 识别为空 | 无语音就停止 | 图标变灰，不粘贴 | < 200ms |
| 退出程序 | 右键→退出 | 图标消失 | < 300ms |

## 六、可访问性

- 热键触发：完全不依赖鼠标 👁️ 视障用户友好
- 托盘图标红/灰切换：色盲用户可通过 tooltip 文本区分状态
- 无音频提示：纯视觉状态（v2 可考虑可选的声音反馈）

---

## 七、不支持的功能（故意的）

- ❌ 暗色模式设置 —— 只有一个图标，没有主题概念
- ❌ 多语言界面 —— About 窗口只有中文
- ❌ 窗口大小调整 —— About 窗口固定大小
- ❌ 动画效果开关 —— 托盘动画极简，对性能无影响
- ❌ 开机自启设置 —— v2 考虑
